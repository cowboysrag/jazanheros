# دليل إعداد الباك-إند (Supabase)

منصة أبطال جازان تستخدم **Supabase** لقاعدة البيانات والمصادقة وتخزين الصور.
الموقع يعمل على بيانات تجريبية حتى تُضبط المفاتيح، وعندها يتحوّل تلقائياً لاستخدام
قاعدة البيانات الحقيقية.

## الخطوات

### 1) أنشئ مشروع Supabase

1. اذهب إلى <https://supabase.com> وسجّل الدخول.
2. **New project** → اختر اسماً (مثل `jazan-heroes`) وكلمة مرور لقاعدة البيانات والمنطقة الأقرب.
3. انتظر اكتمال التهيئة (دقيقة تقريباً).

### 2) أنشئ الجداول

1. من المشروع: **SQL Editor → New query**.
2. انسخ كامل محتوى الملف [`supabase/schema.sql`](../supabase/schema.sql) والصقه.
3. اضغط **Run**. سيُنشئ الجداول وسياسات الأمان وحاويات الصور.

### 3) احصل على المفاتيح

من **Project Settings → API**، انسخ:

- **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
- **anon public key** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### 4) أضف المفاتيح إلى Vercel

في مشروعك على Vercel: **Settings → Environment Variables**، أضف المتغيّرين أعلاه
لكل البيئات (Production / Preview / Development)، ثم أعد النشر (Redeploy).

### 5) (للتطوير المحلي)

```bash
cp .env.example .env.local
# املأ القيم ثم
npm run dev
```

### 6) أنشئ حساب مشرف (اختياري)

بعد تسجيل حسابك الأول من الموقع، اجعله مشرفاً عبر **SQL Editor**:

```sql
update public.profiles set role = 'admin' where email = 'your@email.com';
```

## ما الذي يُفعّله هذا؟

| الميزة | قبل | بعد |
| --- | --- | --- |
| التسجيل/الدخول | وهمي (المتصفح) | حسابات حقيقية |
| البيانات | ثابتة تجريبية | من قاعدة البيانات |
| الصور | عناصر نائبة | رفع حقيقي |
| الداشبورد/الأدمن | عرض فقط | حفظ وإدارة فعليان |
